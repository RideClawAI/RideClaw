## 产品介绍：龙虾出行 Skill

**龙虾出行 Skill** 是一个基于命令行的全流程打车服务助手。当用户使用自然语言表达打车需求（如“我要从北京西站打车去望京”）时，它会被触发并在终端中完成叫车流程。

该产品主要包含以下四个核心功能模块：

### 1. 账号与配置管理

提供基础的用户信息配置功能，类似打车 App 的注册与设置：

- **初始化 (`init`)**：首次使用时绑定用户手机号，可设置“家”和“公司”的常用地址，系统会自动将地址转换为地图坐标。
- **查看配置 (`profile`)**：随时查看当前绑定的个人信息及常用地址坐标。

### 2. 地图与位置服务

为打车提供精确的起终点定位服务：

- **地点搜索 (`location-search`)**：输入关键词和城市，查询具体的标准地址和经纬度坐标。
- **周边搜索 (`place-around`)**：基于指定坐标，搜索附近的餐厅、酒店等兴趣点（POI）。

### 3. 打车核心链路

整个产品最核心的交易与派单流程：

- **预估价格 (`estimate`)**：输入起点和终点后，计算并返回不同车型（如特惠快车、专车等）的预估价格和预估流程 ID。默认推荐特惠快车，用户可选择查看更多车型。
- **支付与下单 (`pay` & `pay-status`)**：采用**”先生成支付订单，再创建打车订单”**的逻辑。系统先返回支付链接，用户付款后，通过 `pay-status` 查询支付状态。**重要**：一旦支付成功（pay_status=2），后续所有查询必须切换到 `query-order` 或 `driver-location`，严禁继续调用 `pay-status`。

**状态转换规则：**
```
pay → pay-status（确认支付）
            ↓ 支付成功（pay_status=2）
      query-order / driver-location（查看司机和行程）
            ↓ 司机已接单
      driver-location（持续跟踪行程）
```

### 4. 订单与行程管理

提供完善的行程追踪与售后服务：

- **订单查询 (`query-order`)**：支付成功后的默认查询命令，用于查看订单状态、司机信息和行程动态。当用户询问"查询"、"什么情况了"、"到哪了"时应使用此命令。
- **司机追踪 (`driver-location`)**：司机接单后，实时获取司机当前位置、距离和预计到达时间。
- **订单详情与取消 (`order-detail` & `cancel-order`)**：行程结束后查看完整的计费详情；在必要时支持随时取消订单。

**订单状态说明：**
- 1-待支付：等待用户支付
- 2-已支付：可查询司机匹配状态
- 3-履约中：行程进行中，可查询司机位置和行程动态
- 4-已完成：行程结束
- 5-已取消：订单已取消
- 6-异常：订单异常

***

**💡 设计亮点说明：**

- **流程解耦**：将复杂的“打车”动作拆分为搜索、预估、支付、查单等独立命令，便于代码维护和渐进式引导用户操作。
- **支付前置安全逻辑**：作为命令行工具，采用先支付预估款再派单的模式，有效规避了“逃单”风险及复杂的免密代扣授权问题，在业务架构上更为严谨安全。
